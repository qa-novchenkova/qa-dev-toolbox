# 🧰 QA Dev Toolbox

React-приложение с QA-инструментами, автотестами, сборкой через **GitHub Actions** и автоматическим деплоем на **GitHub Pages**.

Данное приложение я создала для своего портфолио QA-инженера. Оно показывает полный цикл: локальная разработка, проверка тестами, сборка и публикация сайта.

## Демо

Сайт опубликован через **GitHub Pages**:

https://qa-novchenkova.github.io/qa-dev-toolbox/

## О проекте

**QA Dev Toolbox** — это набор мини-инструментов для повседневной QA-практики:

* генератор баг-репорта в формате Markdown;
* генератор чек-листов для разных типов проверок;
* генератор тестовых данных;
* форматирование и валидация JSON;
* проверка регулярных выражений на тестовом тексте.

Проект сделан как статический сайт, который можно собрать локально и опубликовать через **GitHub Pages**.

## Что отрабатывалось

В ходе работы отрабатывала:

* создание frontend-проекта на **Vite** (инструмент для локального запуска и сборки веб-приложения);
* работу с компонентами **React** (отдельные части интерфейса: вкладки, формы, кнопки, модальное окно);
* вынесение логики в отдельные функции (генерация отчётов, чек-листов, тестовых данных, проверка JSON и regex);
* написание unit- и UI-тестов на **Vitest** и **Testing Library** (проверка отдельных функций и базовых действий пользователя в интерфейсе);
* локальную проверку проекта через `npm test`, `npm run dev` и `npm run build`;
* настройку workflow для **GitHub Actions**;
* автоматический деплой сайта на **GitHub Pages**.

## Инструменты на странице

### Bug Report Builder

Форма для составления баг-репорта. Пользователь заполняет заголовок, окружение, severity, priority, шаги воспроизведения, ожидаемый и фактический результат. На выходе получается готовый Markdown, который можно скопировать в Jira, GitHub Issues или другой таск-трекер.

### QA Checklist Generator

Генерирует чек-лист по выбранной категории:

* UI;
* forms;
* API;
* auth;
* responsive.

### Test Data Generator

Создаёт тестовые данные:

* email;
* password;
* UUID;
* номер телефона;
* дату;
* JSON-объект.

### JSON Formatter / Validator

Проверяет валидность JSON и форматирует его с отступами.

### Regex Tester

Позволяет проверить регулярное выражение на тестовом тексте и увидеть найденные совпадения.

## Тесты

В проекте есть проверки для:

* генерации Markdown-баг-репорта;
* генерации чек-листа;
* генерации тестовых данных;
* валидации JSON;
* поиска совпадений по regex;
* базового поведения интерфейса.

### Пример теста: генерация баг-репорта

Один из unit-тестов проверяет функцию `generateBugReport`.

Логика теста:

1. В функцию передаются тестовые данные: заголовок, окружение, severity, priority, шаги, ожидаемый и фактический результат.
2. Функция собирает из этих данных Markdown-баг-репорт.
3. Тест проверяет, что в результате есть нужные части: заголовок, нумерованные шаги и секция с фактическим результатом.

Пример проверки:

```js
expect(report).toContain('# Bug Report: Кнопка входа не работает')
expect(report).toContain('1. Открыть страницу входа')
expect(report).toContain('2. Нажать кнопку Login')
expect(report).toContain('## Фактический результат')
```

Такой тест помогает убедиться, что при изменениях в коде не сломается основной сценарий генерации баг-репорта.

Запуск тестов:

```bash
npm test
```

Проверка production-сборки:

```bash
npm run build
```

## GitHub Pages

Workflow в `.github/workflows/deploy.yml`:

* устанавливает зависимости;
* запускает тесты;
* собирает проект;
* публикует папку `dist` на **GitHub Pages** после push в ветку `main`.

Последовательность процесса:

```text
локальные изменения → commit → push → GitHub Actions → tests → build → GitHub Pages
```

В `vite.config.js` для GitHub Pages указан base path:

```js
base: process.env.GITHUB_ACTIONS ? '/qa-dev-toolbox/' : '/'
```

Если репозиторий будет называться иначе, нужно изменить `/qa-dev-toolbox/` на актуальное имя репозитория.
