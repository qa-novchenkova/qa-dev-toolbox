    # QA Dev Toolbox

Проект для портфолио QA-инженера.

Проект помогает отработать связку **React + Vite + Vitest + GitHub Actions + GitHub Pages** и одновременно содержит небольшие инструменты, полезные для тестировщика и разработчика.

## О проекте

**QA Dev Toolbox** — это набор мини-инструментов для повседневной QA-практики:

* генератор баг-репорта в формате Markdown;
* генератор чек-листов для разных типов проверок;
* генератор тестовых данных;
* форматирование и валидация JSON;
* проверка регулярных выражений на тестовом тексте.

Проект сделан как статический сайт, который можно собрать локально и опубликовать через **GitHub Pages**.

## Что отрабатывалось

В ходе работы отрабатывала:

* создание frontend-проекта на **Vite**;
* работу с компонентами **React**;
* вынесение логики в отдельные функции;
* написание unit- и UI-тестов на **Vitest** и **Testing Library**;
* проверку проекта через `npm test` и `npm run build`;
* настройку workflow для **GitHub Actions**;
* автоматический деплой сайта на **GitHub Pages**.

## Инструменты на странице

### Bug Report Builder

Форма для составления баг-репорта. Пользователь заполняет заголовок, окружение, severity, priority, шаги воспроизведения, ожидаемый и фактический результат. На выходе получается готовый Markdown, который можно скопировать в Jira, GitHub Issues или другой таск-трекер.

### QA Checklist Generator

Генерирует чек-лист по выбранной категории:

* UI;
* forms;
* API;
* auth;
* responsive.

### Test Data Generator

Создаёт тестовые данные:

* email;
* password;
* UUID;
* российский номер телефона;
* дату;
* JSON-объект.

### JSON Formatter / Validator

Проверяет валидность JSON и форматирует его с отступами.

### Regex Tester

Позволяет проверить регулярное выражение на тестовом тексте и увидеть найденные совпадения.

## GitHub Pages

Workflow в `.github/workflows/deploy.yml`:

* устанавливает зависимости;
* запускает тесты;
* собирает проект;
* публикует папку `dist` на **GitHub Pages** после push в ветку `main`.

В `vite.config.js` для GitHub Pages указан base path:

```js
base: process.env.GITHUB_ACTIONS ? '/qa-dev-toolbox/' : '/'
```

Если репозиторий будет называться иначе, нужно изменить `/qa-dev-toolbox/` на актуальное имя репозитория.
